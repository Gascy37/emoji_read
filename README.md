emoji阅读器插件

这是一个为 MT 插件平台设计的智能文本美化插件，能够根据文本情感和内容自动添加 Emoji 和颜文字（Kaomoji），支持多种风格模式和丰富的自定义配置。

✨ 主要功能

🎨 多种风格模式

· 正常 Emoji 和颜文字：平衡添加，自然表达
· 少量/多量 Emoji：根据需求调节 Emoji 密度
· 仅 Emoji：纯 Emoji 增强表达
· 少量/多量颜文字：侧重日式颜文字风格
· 混合模式：Emoji 与颜文字比例可调
· 纯文本模式：仅调用 AI 处理，不添加符号
· 可爱女仆风格：特殊情境下的撒娇风格处理

⚙️ 智能处理能力

· 智能分块处理：自动分割长文本，支持并发处理
· 代码检测与屏蔽：自动识别代码片段，避免误处理
· 标点智能修正：自动转换中英文标点
· 意境增强：提升文本情感表达能力
· 简洁处理：优化文本表达，避免冗长

🔧 高度可配置

· 多模型支持：集成 Groq、Llama、GPT-OSS、通义千问等 10+ AI 模型
· API 配置灵活：支持内置密钥与自定义 API
· 参数精细调节：温度、Token 数、超时、重试等全面可调
· 实验性功能：支持扩展 Token 上限至 65536

📦 安装与使用

环境要求

· Android 系统
· MT 插件平台

快速开始

1. 下载插件文件（emojiPreference.java 和 main.java）
2. 在 MT 插件平台中导入插件
3. 进入插件设置，根据需要配置 API 和模型参数
4. 选择目标语言（即处理风格），开始使用

⚙️ 配置说明

API 配置

· 使用内置 API：默认开启，无需额外配置
· 自定义 API：支持 Groq 或其他兼容 OpenAI API 的服务

模型选择

· 推荐模型：Llama 3.3 70B 全能版
· 轻量模型：Llama 3.1 8B 即时版
· 安全模型：Llama Guard 系列
· 专业模型：通义千问、Kimi K2 等

处理设置

· 分块大小：100-5000 字符（默认 2000）
· 温度参数：0.0-2.0（默认 0.7）
· 最大 Token：100-8192（实验性支持 65536）
· 并发请求：1-5 个（默认 3）

🛠️ 技术实现

核心特性

· 多线程并发处理：提升长文本处理效率
· 智能错误恢复：自动重试与超时处理
· 正则表达式优化：高效代码检测与字符串处理
· 动态提示词生成：根据模式和功能智能生成 AI 指令

代码结构

· emojiPreference.java：设置界面构建与配置管理
· main.java：核心翻译引擎与 AI 交互逻辑

📝 使用示例

示例输入

```
今天天气真好，我想去公园散步。
```

不同模式输出示例

· 正常模式：今天天气真好☀️，我想去公园散步🚶‍♂️。
· 可爱女仆风格：主人…今天天气真好呢～人家想去公园散步啦…(〃ﾉωﾉ)
· 仅 Emoji：今天天气真好☀️，我想去公园散步🚶‍♂️💕。
· 问题分析模式：这段文本表达了作者在好天气下想去公园散步的愉快心情…

⚠️ 注意事项

1. API 限制：内置 API 有频率限制，建议申请自己的 API 密钥
2. 处理时间：长文本可能需要较长时间处理
3. 代码处理：默认屏蔽代码处理，可在设置中调整
4. 实验功能：扩展 Token 上限可能增加 API 成本
